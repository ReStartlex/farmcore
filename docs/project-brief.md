## 1. Продуктовый анализ

### 1.1. Что продаём (суть оффера)
Продажа **аккаунтов Steam/CS2 для фермы дропа** + **сопровождение под ключ**: помощь с расчётом
объёма под бюджет, объяснение механики, рекомендации по запуску и поддержка после покупки.
Деньги берём за аккаунты; помощь и расчёт — **часть ценности покупки**, а не платные опции.

### 1.2. Экономика (внутренняя, для логики, не для показа дословно)
- Себестоимость аккаунта: ~1100 ₽
- Желаемая маржа: ~500 ₽/акк
- Цена для клиента (tier-логика):
  - 10–29 акк → **1600 ₽/акк**
  - 30–99 акк → **1550 ₽/акк**
  - 100+ акк → **1500 ₽/акк**
- Минимальный заказ: **10 аккаунтов**
- Шаг калькулятора: **5**
- Максимум калькулятора: **300**

### 1.3. Целевая аудитория
Обычные люди, ищущие способ заработка на дропе CS2 без глубокого технического бэкграунда.
Боли: «сколько стоит старт», «когда отобьётся», «сколько брать аккаунтов», «не развод ли это»,
«я ничего не понимаю в CS2 — справлюсь ли». Ищут по запросам: ферма cs2 / кс2 / стим, ферма
под ключ, ферма cs2 2026, купить ферму cs2, окупаемость, заработок на дропе cs2 и т.д.

### 1.4. Главный психологический рычаг
**Эффект масштаба:** чем больше аккаунтов, тем ниже цена за аккаунт и тем больше еженедельных
попыток поймать дроп (включая шанс на дорогой кейс при обновлениях). Это honest-up-sell:
человек сам видит в калькуляторе, что больший объём выгоднее по окупаемости.

### 1.5. Конкурентное отличие (почему мы лучше референса)
| Референс (BossFarm и пр.) | Наш продукт |
|---|---|
| Тёмный «ИИ-фиолетовый» шаблон | Светлый, чистый, дорогой UI с фирменным акцентом |
| Калькулятор вокруг софта/панели/курса доллара | Калькулятор только вокруг аккаунтов, бюджета, дохода и окупаемости — проще и честнее |
| Безличный бренд | Личная экспертиза «в Steam с 2014, со времён операции Bravo» |
| Жёсткие «тарифы» | Гибкие **сценарии запуска** + персональный подбор под бюджет |
| Обещания дохода | Аккуратные формулировки: «ориентировочно», «средний сценарий», «зависит от дропа» |

### 1.6. Тон и ограничения (комплаенс копирайтинга)
**Нельзя:** «гарантированный доход», «100% окупаемость», «без рисков», «пассивный доход без
усилий», «стабильная прибыль гарантирована».
**Можно:** «ориентировочный расчёт», «средний сценарий», «может отличаться», «зависит от дропа,
рынка и количества аккаунтов», «чем больше аккаунтов — тем выше шанс поймать дорогой дроп»,
«помогаю подобрать объём под бюджет», «перед покупкой объясню механику».
Дисклеймеры присутствуют, но **ненавязчиво** (мелкая сноска под калькулятором, строка в FAQ),
чтобы не убивать желание купить.

## 3. Визуальная концепция (премиум, светлая)

### 3.1. Настроение
«Дорогой финтех + игровой акцент». Светло, воздушно, много белого пространства, мягкие тени,
крупная уверенная типографика. Один яркий фирменный акцент на CTA и ключевых цифрах. Никакого
«тёмного неонового ИИ-шаблона».

### 3.2. Палитра (light theme)
- Фон: `#F7F8FB` (почти белый, тёплый), секции-карточки: `#FFFFFF`.
- Текст основной: `#0E1116`, вторичный: `#5B6472`.
- Акцент (бренд): глубокий индиго-фиолетовый, но **благородный**, не кислотный:
  `#5B5BD6` → градиент к `#7A6CF0`. (Сохраняем «фиолетовую» нишевую узнаваемость, но делаем
  её дорогой на светлом фоне.)
- Доп. акцент успеха/денег: спокойный изумруд `#12A594` (для блоков дохода/окупаемости).
- Линии/границы: `#E6E8EE`.
- Тени: мягкие, многослойные, низкий contrast (`0 8px 30px rgba(16,17,22,.06)`).

> Опционально предложу заказчику тёмную тему как переключатель, но **дефолт — светлая**.

### 3.3. Типографика
- Заголовки: гротеск с характером — **Unbounded** или **Manrope** (display weight).
- Текст: **Inter** / **Manrope** (читабельный, премиум-нейтральный).
- Цифры в калькуляторе: tabular-nums, крупные, с лёгкой анимацией счётчика (count-up).

### 3.4. Премиум-детали и 3D
- Hero: лёгкая 3D/parallax-композиция — стилизованные «карточки аккаунтов» / кейсы CS2,
  парящие с мягким bloom, реализовано как **CSS/SVG + framer-motion** (легко) или
  **react-three-fiber** (если хотим «вау» 3D). По умолчанию — лёгкий вариант ради скорости и SEO;
  тяжёлое 3D — опционально и lazy-loaded.
- Стеклянные карточки (subtle glassmorphism) только на акцентных блоках, без перегруза.
- Микроанимации появления (intersection observer), hover-подъём карточек, плавный скролл к якорям.
- Декор: тонкие сетки/линии, мягкие градиентные «пятна» света на фоне (очень деликатно).

### 3.5. Доступность
Контраст AA, focus-стейты, prefers-reduced-motion, семантические теги, alt у изображений.

## 7. Логика калькулятора (точная спецификация)

### 7.1. Входные параметры
- `accountsCount`: целое, **min 10, max 300, шаг 5** (слайдер + кнопки −/+ + ручной ввод).

### 7.2. Цена за аккаунт (tier-логика)
```
if (accountsCount >= 100) accountPrice = 1500
else if (accountsCount >= 30) accountPrice = 1550
else accountPrice = 1600
```

### 7.3. Коэффициент масштаба
```
if (accountsCount >= 100) scaleBonus = 1.15
else if (accountsCount >= 30) scaleBonus = 1.08
else scaleBonus = 1.00
```

### 7.4. Формулы
```
startBudget        = accountsCount * accountPrice
monthlyIncomeMin   = accountsCount * 70  * 4 * scaleBonus
monthlyIncomeMax   = accountsCount * 100 * 4 * scaleBonus
monthlyIncomeBase  = accountsCount * 400 * scaleBonus      // средний сценарий

paybackMonthsBase  = startBudget / monthlyIncomeBase       // по среднему
paybackMonthsMin   = startBudget / monthlyIncomeMax        // быстрый край
paybackMonthsMax   = startBudget / monthlyIncomeMin        // медленный край
```

### 7.5. Округление
- Деньги → до целых рублей, формат с пробелом-разделителем тысяч: `16 000 ₽`.
- Окупаемость → до 1 знака после запятой: `4.0 мес`, `3.6 мес`.

### 7.6. Что показываем в UI
- Цена за аккаунт.
- «Нужно на старт» = `startBudget`.
- «Ориентир по дропу / мес» = диапазон `monthlyIncomeMin–monthlyIncomeMax`.
- «Средний сценарий / мес» = `monthlyIncomeBase`.
- «Когда может отбиться» = `paybackMonthsBase` мес (опц. с диапазоном min–max мелко).
- Динамическая подсказка про следующий tier (до 30 / до 100).

### 7.7. Контрольные значения (ПРОВЕРЕНО — все сходятся ✅)
| Акк | Цена | Старт | Доход/мес (min–max) | Средний | Окуп. (база) |
|----:|-----:|------:|--------------------:|--------:|------:|
| 10  | 1600 | 16 000 ₽  | 2 800 – 4 000 ₽   | 4 000 ₽  | 4.0 мес |
| 30  | 1550 | 46 500 ₽  | 9 072 – 12 960 ₽  | 12 960 ₽ | 3.6 мес |
| 50  | 1550 | 77 500 ₽  | 15 120 – 21 600 ₽ | 21 600 ₽ | 3.6 мес |
| 100 | 1500 | 150 000 ₽ | 32 200 – 46 000 ₽ | 46 000 ₽ | 3.3 мес |

### 7.8. ⚠️ Важное наблюдение и рекомендация по доработке
По формулам `monthlyIncomeBase == monthlyIncomeMax` (оба `= count * 400 * scaleBonus`),
поэтому «средний сценарий» совпадает с верхней границей диапазона, а `paybackBase == paybackMin`.
Это слегка оптимистично и снижает достоверность.

**Рекомендую** (на согласование заказчика) сделать средний = середина диапазона:
```
monthlyIncomeBase = accountsCount * 85 * 4 * scaleBonus   // среднее между 70 и 100
```
Тогда для 10 акк средний = 3 400 ₽, окупаемость ≈ 4.7 мес — выглядит честнее и не противоречит
тону «средний сценарий». **По умолчанию реализуем по ТЗ (контрольные значения выше), а этот
вариант оставляю как опцию-переключатель в конфиге.**

### 7.9. CTA-интеграция калькулятора
Кнопка формирует Telegram deep link с предзаполненным текстом, напр.:
`https://t.me/assistent_cs?text=Привет!%20Хочу%20рассчитать%20ферму%20на%20{N}%20аккаунтов...`
Параллельно значения пишутся в форму заявки (скрытые поля), чтобы лид не терялся.

### 8.4. FAQ (вопросы + опорные ответы, под SEO и возражения)
1. **Что такое CS2 ферма?** — Несколько аккаунтов Steam, которые получают еженедельный дроп
   предметов/кейсов CS2; их можно продавать на рынке.
2. **Сколько стоит ферма CS2?** — От 16 000 ₽ за старт (10 акк × 1600 ₽). Цена за аккаунт падает
   при большем объёме (от 1500 ₽).
3. **Сколько аккаунтов нужно для старта?** — Минимум 10. Оптимально 30–75 для стабильности.
4. **Почему чем больше аккаунтов, тем выгоднее?** — Ниже цена за аккаунт + больше попыток на дроп
   и выше шанс поймать дорогой предмет.
5. **Сколько может приносить один аккаунт?** — Ориентир ~70–100 ₽/нед; часто считают ~400 ₽/мес.
   Зависит от дропа и рынка.
6. **За сколько окупается ферма CS2?** — По среднему сценарию ориентир ~3.3–4 мес; может
   отличаться, не гарантия.
7. **Можно ли начать с 10 аккаунтов?** — Да, это минимальный заказ.
8. **Что если у меня конкретный бюджет?** — Назовите сумму — подберу максимум аккаунтов и посчитаю.
9. **Можно ли докупить аккаунты позже?** — Да, объём наращивается.
10. **Есть ли гарантия дохода?** — Нет. Это ориентировочный расчёт; итог зависит от дропа и рынка.
11. **Что я получу после покупки?** — Аккаунты, рекомендации по запуску, сопровождение и связь.
12. **Как связаться для расчёта?** — Telegram @assistent_cs — посчитаем под ваш бюджет.
